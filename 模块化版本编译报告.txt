================================================================================
                              编译工具
================================================================================

1. 手动编译命令
---------------

# 编译单个模块
g++ -std=c++14 -Wall -Wextra -c \
    -ID:\eigen3 -DSIMTOOLS_STATIC -DUSE_EIGEN \
    SimTools_XXX.cpp -o SimTools_XXX.o

# 创建静态库
ar rcs libSimTools.a SimTools_*.o

# 链接生成可执行文件
g++ -std=c++14 -Wall -Wextra \
    -ID:\eigen3 -DSIMTOOLS_STATIC -DUSE_EIGEN \
    your_program.cpp libSimTools.a -o your_program.exe

2. 使用 Makefile（推荐）
-----------------------

cd E:\MyCode\SimTools_v2

mingw32-make          # 编译测试程序
mingw32-make lib      # 只编译静态库
mingw32-make run      # 编译并运行
mingw32-make clean    # 清理编译产物
mingw32-make rebuild  # 重新编译

Makefile 特点：
--------------
✅ 自动编译所有模块
✅ 增量编译（只重新编译修改过的文件）
✅ 支持并行编译（mingw32-make -j4）
✅ 清晰的编译进度显示
✅ 自动生成静态库

                              常见问题
================================================================================

Q1: 如何只使用部分模块？
-------------------------
A: 只链接需要的 .o 文件，例如：
   g++ your_program.cpp \
       SimTools_Coordinate.o \
       SimTools_Atmosphere.o \
       -o your_program.exe

Q2: 如何创建自己的程序？
-------------------------
A: 参考test_modular.cpp的结构：

   #include "SimTools_v2.h"
   #include <iostream>

   int main() {
       using namespace SimTools;

       auto air = Atmosphere::GetParameters(10000);
       std::cout << "温度: " << air.temperature << std::endl;

       return 0;
   }

   编译：
   g++ your_program.cpp libSimTools.a -o your_program.exe

Q3: 如何调试单个模块？
---------------------
A: 单独编译该模块并查看警告：
   g++ -std=c++14 -Wall -Wextra -c \
       -ID:\eigen3 -DSIMTOOLS_STATIC -DUSE_EIGEN \
       SimTools_Coordinate.cpp -o SimTools_Coordinate.o

Q4: 能否混合使用两种版本？
-------------------------
A: 不能！它们提供相同的功能，但使用方式不同：
   - 模块化版本：链接多个 .o 文件或静态库
   - 单文件版本：只编译一个 .cpp 文件

   选择其中一种即可。

Q5: 中文乱码怎么办？

   在Windows cmd中也能正常显示中文。
