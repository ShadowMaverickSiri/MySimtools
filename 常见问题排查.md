# Visual Studio 编译问题排查指南

## 问题：单独编译 .cpp 文件报错

### 常见错误类型：

#### 1. 编码错误 (C4819)

```
error C4819: 文件包含不能在当前代码页中表示的字符
```

**解决方案**：已在 CMakeLists.txt 中添加 `/utf-8` 编译选项

#### 2. 缺少预处理器定义

**检查项目属性**：

- 右键项目 → 属性 → C/C++ → 预处理器
- 确保包含 `SIMTOOLS_STATIC`

#### 3. 找不到 Eigen3 头文件

**检查项目属性**：

- 右键项目 → 属性 → C/C++ → 常规 → 附加包含目录
- 应包含 `D:\eigen3` 或你的 Eigen3 路径

### 快速验证步骤：

1. **在 Visual Studio 中编译整个解决方案**
   
   - 不要单独编译 .cpp 文件
   - 使用：生成 → 生成解决方案 (Ctrl+Shift+B)

2. **查看输出窗口**
   
   - 查看 "输出" 窗口而不是 "错误列表"
   - 输出 → 显示输出从：生成

3. **如果仍有错误，运行以下命令诊断**：
   
   ```cmd
   cd E:\MyCode\SimTools_v2\build
   cmake .. -G "Visual Studio 16 2019"
   cmake --build . --config Release
   ```

### 替代方案：使用命令行编译（推荐）

如果 Visual Studio 仍有问题，可以使用 MinGW 编译：

```cmd
cd E:\MyCode\SimTools_v2
g++ -std=c++14 -ID:\eigen3 -DSIMTOOLS_STATIC ^
    SimTools_Interpolation.cpp ^
    SimTools_Coordinate.cpp ^
    SimTools_Geodesy.cpp ^
    SimTools_Atmosphere.cpp ^
    SimTools_Random.cpp ^
    SimTools_FileIO.cpp ^
    SimTools_Numerical.cpp ^
    SimTools_Geometry.cpp ^
    SimTools_MatrixUtils.cpp ^
    SimTools_Time.cpp ^
    SimTools_Simulation.cpp ^
    -c
```

这会生成所有的 .obj 文件，可以验证编译是否正常。

### 预期的编译输出

成功编译后应该在 `build\Debug` 或 `build\Release` 目录看到：

- `SimTools_static.lib` (静态库)
- `SimTools_example.exe` (示例程序)
- `SimTools_test.exe` (测试程序)
