# SimTools v2.0 - 编译器兼容性说明

## 📋 使用的C++标准

**SimTools v2.0 使用 C++14 标准**

- **CMake设置**: `CMAKE_CXX_STANDARD 14`
- **编译选项**: `-std=c++14` (GCC/Clang) 或 `/std:c++14` (MSVC)

## 🔍 代码中使用的C++11/14特性

### 1. **auto 关键字** (类型推导)
```cpp
auto air = Atmosphere::GetParameters(10000);
auto now = std::chrono::system_clock::now();
auto it = std::upper_bound(xx.begin(), xx.end(), x);
```
- **引入**: C++11
- **VS2010**: ❌ 不支持
- **VS2013**: ✅ 支持
- **VS2015+**: ✅ 完全支持

### 2. **constexpr** (常量表达式)
```cpp
constexpr double GPS_TO_UNIX_OFFSET = 315964800.0;
constexpr double SECONDS_PER_WEEK = 7 * 24 * 3600.0;
```
- **引入**: C++11
- **VS2010**: ❌ 不支持
- **VS2013**: ⚠️ 部分支持
- **VS2015+**: ✅ 完全支持

### 3. **Lambda 表达式**
```cpp
auto f = [](double x) { return x * x - 4.0; };
auto df = [](double x) { return 2.0 * x; };
```
- **引入**: C++11
- **VS2010**: ❌ 不支持
- **VS2013**: ✅ 支持
- **VS2015+**: ✅ 完全支持

### 4. **Range-based for 循环**
```cpp
for (const auto& row : data) {
    // 处理数据
}
```
- **引入**: C++11
- **VS2010**: ❌ 不支持
- **VS2013**: ✅ 支持
- **VS2015+**: ✅ 完全支持

### 5. **std::chrono** (时间库)
```cpp
auto now = std::chrono::system_clock::now();
auto duration = now.time_since_epoch();
```
- **引入**: C++11
- **VS2010**: ❌ 不支持
- **VS2013**: ⚠️ 部分支持（有些bug）
- **VS2015+**: ✅ 完全支持

### 6. **initializer_list**
```cpp
std::vector<double> x = {0, 1, 2, 3, 4, 5};
```
- **引入**: C++11
- **VS2010**: ❌ 不支持
- **VS2013**: ✅ 支持
- **VS2015+**: ✅ 完全支持

### 7. **std::array**
```cpp
std::array<int, 3> arr = {1, 2, 3};
```
- **引入**: C++11
- **VS2010**: ❌ 不支持
- **VS2013**: ✅ 支持
- **VS2015+**: ✅ 完全支持

### 8. **右值引用与移动语义**
```cpp
Vector3d(Vector3d&& other) noexcept; // 移动构造函数
```
- **引入**: C++11
- **VS2010**: ⚠️ 部分支持（有bug）
- **VS2013**: ✅ 支持
- **VS2015+**: ✅ 完全支持

### 9. **noexcept** (异常规范)
```cpp
inline Vector3d operator-(const Vector3d& other) const noexcept;
```
- **引入**: C++11
- **VS2010**: ❌ 不支持
- **VS2013**: ✅ 支持
- **VS2015+**: ✅ 完全支持

## ❌ VS2010 的主要问题

### 1. **缺少 `auto` 类型推导**
所有使用 `auto` 的地方都需要显式指定类型：
```cpp
// 当前代码 (VS2015+)
auto air = Atmosphere::GetParameters(10000);

// VS2010 需要
Atmosphere::Parameters air = Atmosphere::GetParameters(10000);
```

### 2. **不支持 Lambda 表达式**
```cpp
// 当前代码 (VS2015+)
auto f = [](double x) { return x * x; };

// VS2010 需要使用函数对象
struct SquareFunc {
    double operator()(double x) const { return x * x; }
};
SquareFunc f;
```

### 3. **不支持 Range-based for**
```cpp
// 当前代码 (VS2015+)
for (const auto& row : data) { }

// VS2010 需要
for (std::vector<double>::const_iterator it = data.begin();
     it != data.end(); ++it) { }
```

### 4. **不支持 `std::chrono`**
VS2010 需要使用第三方时间库或系统API。

### 5. **不支持 `constexpr`**
```cpp
// 当前代码 (VS2015+)
constexpr double SECONDS_PER_WEEK = 7 * 24 * 3600.0;

// VS2010 需要
const double SECONDS_PER_WEEK = 7 * 24 * 3600.0; // 运行时常量
```

### 6. **C++标准库不完整**
VS2010 的 `<vector>`, `<array>`, `<functional>` 等头文件缺少C++11特性。

## 💡 推荐方案

### 对于 VS2010 用户

**方案 1：升级编译器（强烈推荐）**
- ✅ 下载免费的 **Visual Studio 2019 Community** 或 **VS2022 Community**
- ✅ 或者使用 **Build Tools for Visual Studio 2019**（命令行工具）
- ✅ 这些都是免费的，完全支持C++14

**方案 2：使用 MinGW-w64**
- ✅ 安装 MinGW-w64（包含 g++ 编译器）
- ✅ 完全支持 C++14
- ✅ 与VS2010可以共存

**方案 3：修改代码（不推荐）**
- ❌ 工作量巨大
- ❌ 需要重写所有 `auto`, Lambda, `std::chrono` 等
- ❌ 维护困难

## ⚙️ CMake 配置

### 生成 VS2019 项目
```bash
cmake -G "Visual Studio 16 2019" -A x64 ..
```

### 生成 VS2015 项目
```bash
cmake -G "Visual Studio 14 2015" -A x64 ..
```

### 生成 VS2013 项目（不推荐）
```bash
cmake -G "Visual Studio 12 2013" -A x64 ..
```
⚠️ **注意**: VS2013 会生成项目，但编译会失败（缺少C++14特性）

### VS2010 不可用
❌ **CMake 不支持生成 VS2010 项目用于 C++14**
